# Loreweaver 路线图

Loreweaver 年轻，主要由一个人在 AI 协助下开发。这是诚实的前进计划——项目当前的专注点、随后的更大方向，以及一个值得公开决策的开放设计问题。关于层级契约和铁律，请参见 [CLAUDE.md](../CLAUDE.md)。

## 志向

Loreweaver 的目标不是"用 AI 替代一个跑团主持人"，而是做 **RPG 领域的 Claude Code**。强大的 coding agent 已经很多，合格的 RPG agent 却太少太少。这里的每一层——真骰子和硬规则、靠工具调用行事的守秘人、只知道自己该知道的事的子角色、照 SKILL.md 与酒馆规范生长的扩展生态——都朝着同一个方向：让"跑好一个世界"成为 agent 的一等能力。

## 现状

确定性引擎——骰子（基于 `d20`）、CoC/DnD 成功等级、角色数学、规则验证、游戏时钟——是稳固的核心,由确定性、离线测试套件覆盖。**终端(OpenTUI)客户端是唯一重心**,经 **Iroh** p2p 传输连接(拨一个 ticket——无需域名/TLS/端口转发)。聊天平台适配器(Discord · Telegram · QQ · 飞书)**代码仍在树内,但无人维护、未对真平台实测**——保留在网关传输无关的接缝后面,以便*将来*能复活,但当前不作承诺。(React 网页客户端、WebSocket 服务路径、Docker 部署、SSH 客户端都已移除,把维护面收窄到一个人真能测的范围;WebSocket 仅作离线测试传输保留。富媒体现在通过同一套房间作用域媒体通道支持自托管图片 handout 和音频 blob。)

## 基础设施——已完成

硬化工作刚刚落地了那些在添加广度前必须正确处理的不起眼的东西——该项目现在可以干净安装、行为正确，对小群体的运行也更安全：

- **可安装性。** Wheel 包含每个软件包和运行时数据（本地化、规则包），所以 `pip install` 可以在干净的环境中工作——而不仅限于源代码签出。
- **权限模型。** 玩家/守护者区分现在在*每个*命令表面执行（之前仅在管理框架上保持——玩家密钥可以通过终端运行守护者专用命令）。暴露秘密的回复——掩盖的 API 密钥、守护者专用的传说——仅限于调用者，不向房间广播。
- **角色正确性。** 编辑技能/属性不再治疗受伤的调查员，创建派生正确的起始生命值（满血/满魔力、SAN = min(POW, SANMAX)）；每个属性设置路径都被限制在规则包范围内。
- **诚实的审核。** 内容过滤器默认关闭且无捆绑词表（可配置），文档明确说明这一点，而不是暗示内置审核。
- **真实模型红线门控。** 每晚作业通过真实（廉价）模型运行转轮管道，在**泄漏率**（守护者秘密的逐字*和*转述哨兵）或**骰子优先错过**（应该已投掷但未投掷的检查）时失败——整个项目所依赖的两个声明的第一个自动守护者。（参见 [下文](#offline-tests-vs-real-model-quality) 了解为什么这与离线套件分离。）
- **传输 + 内务。** WS 握手超时、全局连接上限和可选 TLS（带反向代理指导）；Python 3.11 *和* 3.12 上的 CI；清理了死代码和陈旧引用。

## 近期

- **端到端实时测试一个聊天适配器。** 选择一个（QQ 最完整），针对真实平台驱动它直到它真正工作；将其他标记为实验性，而不是暗示它们已完成。
- **多人游戏抛光。** 既然权限模型已执行，收紧其余网络游玩的粗糙边角（真正的机器人循环守护者、更丰富的迟到加入者状态），使可信人员之间的房间真正舒适。

## 更大方向——世界引擎

差异化因素是冒险*下方*的世界，而不仅仅是带骰子的聊天。长期方向：

- **更深世界书：** 生成型世界（不仅仅是关键字/向量检索传说）、**活生生的因果时间线**，其中事件有后果传播、**规范一致性**使守护者无法矛盾既定事实。
- **迟到加入者赶上：** 在活动中期加入的玩家被赶上他们角色*应该*知道的信息——而不泄漏他们不应该知道的。
- **D&D Beyond 角色单导入**，与现有的 SillyTavern 卡路径并排。
- **CI 发布工件**使版本成为可下载、可安装的东西。

## 开放设计问题——守护者的秘密住在哪里？

如今，守护者的系统提示在（接近）完整中携带模块的守护者池，守护者专用工具逐字将秘密交回模型；守护者方面的反元游戏因此是*纪律*（"不要向玩家引用这个"指示）加上确定性守护者/玩家池分割——对子行为体（仅从它们自己的记录构建）强大，对守护者本身较弱。

另一种方式是将秘密*保持在*基础提示之外，让守护者通过工具按需拉取它们，使模型只会持有它刚推理的特定秘密。这以提示便利和延迟权衡较小泄漏表面。这是真正的架构分叉，值得刻意决策而不是无意漂流——欢迎输入。

## 离线测试 vs. 真实模型质量

值得明确说明，因为绿色 CI 容易被过度阅读：离线套件是确定性的，使用*脚本化*守护者。它严格证明确定性机制——守护者/玩家知识编辑、子行为体提示隔离、真实种子骰子、命令表面——它会捕捉这些中任何一个的回归。它**不能**证明实时模型避免泄漏显示给它的秘密，或它投掷后叙述；那些是模型行为属性，它们正是真实模型红线门（现在每晚运行）存在以测量的。读取"CI 绿色"为*引擎正确*，不是*守护者好*。
